Formalising Patterns with Tables

10 minutes · Position number → Term value
What goes in a table?

A pattern table has two rows:

  • Position (n) — which term it is: 1st, 2nd, 3rd…
  • Term Value — the actual number in the sequence at that position.
The goal: find a single rule linking n to the term value. Then you can calculate any term without writing them all out!
Example: 5, 10, 15, 20…
Position (n)Term Value
15
210
315
420
"Multiply the position number by 5" 5 × n
So the 10th term = 5 × 10 = 50 — no need to list all terms!
Two-step rules
Some patterns need two operations — multiply then add (or subtract). Example: 5, 7, 9, 11…
Position (n)Term ValueHow to see the rule
152×1 + 3 = 5 ✓
272×2 + 3 = 7 ✓
392×3 + 3 = 9 ✓
4112×4 + 3 = 11 ✓
"Multiply position by 2, then add 3" 2n + 3
How to find a two-step rule: The difference between consecutive terms tells you the multiplier. Then check what you need to add/subtract to make it work for position 1.
Here: difference = 2, so multiply by 2. Then 2×1 = 2, but we need 5, so add 3.
